## 3.1 データフォーマット

- JSON にデフォルトとして対応し、需要や必要性に応じて XML 等に対応する
- データフォーマットの指定方法の実装
	1. クエリパラメータ (`https://api.example.com/v1/users?format=xml`)
	2. 拡張子 (`https://api.example.com/v1/users.json`)
	3. リクエストヘッダのメディアタイプを指定 (`Accept: application/json`)
- 基本的にはクエリパラメータで実装する
